Setup Guide AI Signals Intelligence Platform — Pipeline v20

Overview

This guide walks through setting up the complete CalmFalcon multi-agent pipeline from scratch. You will configure cloud accounts, import n8n workflows, connect API credentials, run a test pipeline, and enable weekly scheduling. Total setup time: ~30–45 minutes.

What You'll Build

A 4-agent AI pipeline (Theme Selector → Theme Writer ×8 → Verifier → Brief Writer) running weekly in n8n Cloud, ingesting 200+ signals from Exa + GitHub, scoring themes, and delivering a weekly brief stored in Supabase.

Setup Steps

1

Create a Supabase Project

Go to supabase.com → click New Project. Choose a region close to you. Once created, navigate to Project Settings → API and note down:

  • Project URL (looks like https://xxxxx.supabase.co)
  • Service role key (under "Project API keys" — use the service_role one, not anon)
2

Run the Database Schema

In Supabase, go to SQL Editor → New Query. Copy the contents of n8n-workflows/supabase-schema.sql into the editor and click Run. This creates 8 tables:

  • Production (5): pipeline_runs, themes, weekly_briefs, eval_reports, prompt_versions
  • Golden Set (3): golden_set_runs, golden_set_themes, golden_set_eval_scores

Verify by going to the Table Editor — you should see all 8 tables listed.

3

Set Up n8n Cloud

Sign up at n8n.io and create a new workspace. No installation needed — n8n Cloud is a managed SaaS platform. The Starter plan supports up to 5 active workflows, which is enough for this pipeline plus evals.

4

Add Workflow Variables

In n8n, go to Settings → Variables and add the following 5 variables:

OPENAI_API_KEY   = sk-...
EXA_API_KEY      = ...
GITHUB_TOKEN     = ghp_...
SUPABASE_URL     = https://xxxxx.supabase.co
SUPABASE_KEY     = eyJhbGci...  (service_role key)

These will be referenced by the workflow nodes using {{ $vars.VARIABLE_NAME }}.

5

Import the Main Pipeline

In n8n, click the … menu (top right) → Import from File. Select calmfalcon-signals-pipeline-multi-agent.json. Once imported, press Cmd+A then Tidy Up to auto-arrange the 34 nodes for readability.

You should see the full pipeline: Schedule Trigger → Save Prompt Version → Build Search Queries → [Exa + GitHub] → Merge → Process Signals → ... → Brief Writer → Save to Supabase.

6

Import Eval Pipelines

Import each of the following eval workflows separately:

  • calmfalcon-eval-automated.json — 13 deterministic checks (production)
  • calmfalcon-eval-llm-judge.json — GPT-4 Turbo 4-dimension judge (production)
  • calmfalcon-eval-golden-set.json — 9-check golden set scorecard (in progress)

Each eval runs as a separate workflow that reads from Supabase tables populated by the main pipeline.

7

Connect OpenAI Credentials

In the main pipeline, click each LLM node and assign your OpenAI credential:

  • Theme Selector Model
  • Theme Writer Model
  • Verifier Model
  • Brief Model

Also do this for the LLM Judge node in the llm-judge eval workflow. If you don't have an OpenAI credential yet, click Create New and paste your OPENAI_API_KEY.

8

Test Run the Main Pipeline

Click Execute Workflow at the bottom of the canvas. The pipeline takes ~3 minutes to complete. Watch the nodes light up as they execute. When done, verify in Supabase:

SELECT run_id, total_themes, avg_confidence, status
FROM pipeline_runs
ORDER BY completed_at DESC
LIMIT 1;

You should see a row with total_themes = 8 and status = 'completed'. Check the themes table to view the 8 generated themes.

9

Run Production Evals

Execute the automated eval workflow first (deterministic checks, ~10 seconds), then the LLM judge workflow (GPT-4 Turbo scoring, ~30 seconds). Verify results in Supabase:

SELECT eval_type, score, overall_verdict, created_at
FROM eval_reports
ORDER BY created_at DESC
LIMIT 2;

You should see two rows: automated (score like "10/13") and llm_judge (score like "4.3/5.0" with verdict "PUBLICATION READY").

10

Run Golden Set Eval (Optional — In Progress)

Import calmfalcon-golden-set-pipeline.json as a separate workflow. Connect OpenAI credentials to its LLM nodes. Execute it to send 32 curated signals (gs-v2.1) through the full pipeline. Results saved to golden_set_runs + golden_set_themes.

Then run calmfalcon-eval-golden-set.json for the scorecard:

SELECT north_star_pct, notes
FROM golden_set_eval_scores
ORDER BY scored_at DESC
LIMIT 1;

⚠ Status: In progress — Save to Supabase routing currently being debugged. See backlog item #17.

11

Enable Weekly Scheduling

Toggle the main pipeline workflow to Active (top-right switch). The Schedule Trigger node fires every Monday at 6 AM (your n8n workspace timezone). You'll get a fresh brief delivered to Supabase every Monday morning.

To change the schedule, click the Schedule Trigger node and modify the cron expression.

Troubleshooting

IssueCauseFix
OpenAI API errorInvalid key or no GPT-4 Turbo accessVerify key in Variables. Check GPT-4 Turbo access and credits at platform.openai.com/usage.
Exa API errorRate limit or invalid keyVerify EXA_API_KEY. Free tier is 1000 req/day — each run uses 14.
Supabase errorWrong URL or using anon keyCheck URL format (https://xxxxx.supabase.co). Use service_role key, not anon.
GitHub rate limitNo token (60 req/hr default)Add GITHUB_TOKEN to Variables for 5000 req/hr.
0 themes outputVerifier removed all themesCheck Verifier Agent output. May need to increase max_tokens in the LLM node options.
SyntaxError in code nodeEscaped backticks in template literalsCheck the code node for \` sequences — replace with plain backticks.
Theme Writer runs 0 timesSplit Clusters returned empty arrayCheck that Theme Selector output contains valid selected_clusters array with 8 items.
Duplicate theme_idsOld data conflict in Supabase UPSERTEnsure theme_ids include run timestamp suffix (e.g. theme-001-1774413902711).
